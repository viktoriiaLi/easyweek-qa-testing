# Test Types and Levels

This document describes the test types and testing levels applied to the EasyWeek booking system.

## Automated Testing

Automated testing is applied to business-critical and repetitive scenarios to ensure system stability during regression testing.

### Automated Scenarios:
- Prevention of double booking for the same time slot
- Validation of client contact data during booking
- Prevention of booking incompatible services

### Test Levels:
- API testing
- Integration testing

## Manual Testing

Manual testing is used for scenarios that require human judgment or are difficult to automate.

### Manual Scenarios:
- User interface and usability testing
- Validation of error messages and system notifications
- Calendar and schedule visual verification
- Edge case and exploratory testing

### Test Levels:
- System testing
- User acceptance testing (UAT)
